# TeamTrustGate 🛡️

> ИИ-агент для автоматической обработки клиентских запросов от сейлзов в продуктовую команду.

**Стек:** Python 3.11 · Telegram Bot API · Gemini 2.5 Flash · DeepSeek (резерв) · Jira REST API v2 · SQLite · Railway

---

## Зачем это нужно

Сейлзы каждый день получают запросы от клиентов — «хотим фичу X», «не работает Y», «конкуренты уже умеют Z». Раньше это шло в общий чат или в Jira вручную: без структуры, без приоритетов, с дублями.

TeamTrustGate решает это автоматически:

- Сейлз пишет запрос в Telegram своими словами
- ИИ извлекает суть, оценивает бизнес-риск и приоритет
- Бот создаёт структурированный тикет в Jira
- Продуктовая команда получает готовый, оценённый запрос

## Флоу обработки запроса

```
1. Сейлз пишет запрос в Telegram
          │
          ▼
2. LLM анализирует сообщение (extraction.md)
   └── confidence < 0.8? → задаёт уточняющий вопрос (до 3 раундов)
          │
          ▼
3. Дедупликация — сравнивает с бэклогом Jira
   └── дубликат найден? → добавляет комментарий, уведомляет сейлза
          │
          ▼
4. RICE-скоринг (scoring.md)
   └── Reach × Impact × Confidence × Strategy Fit → 0–400 баллов
          │
          ▼
5. Превью тикета сейлзу для подтверждения
   └── [✅ Создать] [✏️ Уточнить] [❌ Отменить]
          │
          ▼
6. Тикет создаётся в Jira
          │
          ▼
7. Polling каждые 5 минут → уведомление при смене статуса
```

## Функциональность

### Обработка запросов
- Приём запросов через Telegram
- Семантический анализ сообщения — очистка от эмоций и сленга сейлзов
- Уточняющие вопросы при недостатке данных (до 3 раундов)
- Фильтрация спама и нерелевантных запросов (`should_reject`)

### Дедупликация
- Батч-сравнение нового запроса со всеми тикетами бэклога за последние N дней
- Семантическое сравнение через LLM (не просто ключевые слова)
- Keyword fallback (Jaccard similarity) при недоступности LLM
- При дубликате — добавление комментария в существующий тикет

### Скоринг (Weighted RICE)
- **Reach** — охват: один клиент / сегмент / все клиенты
- **Impact** — риск потери выручки (1–10)
- **Confidence** — уверенность в данных (0.0–1.0)
- **Strategy Fit** — соответствие продуктовой стратегии (1–10)
- Enterprise-множитель: если `is_enterprise_risk=true` и `revenue_at_risk≥9` → автоповышение приоритета
- Итоговый скор 0–400 → Highest / High / Medium / Low

### Jira интеграция
- Создание структурированных тикетов (Story)
- Автоматические лейблы: `teamtrustgate`, `ai-generated`, `sales-request`
- Смена статуса прямо из Telegram (▶️ В работу / ✅ Готово / 🔄 На проверку / 🚫 Отклонить)
- Удаление тикета с подтверждением
- Просмотр последних 5 тикетов (`/list`)
- Circuit breaker — при 5 ошибках подряд блокирует запросы на 5 минут

### Telegram UX
- Превью тикета перед созданием — сейлз видит что распознал ИИ и подтверждает
- Inline-кнопки под каждым тикетом
- Уведомления при изменении статуса тикета (polling каждые 5 минут)
- Поддержка параллельных сессий (`concurrent_updates=True`)

### Отказоустойчивость
- Автоматический failover: Gemini → DeepSeek при недоступности
- Retry с экспоненциальным backoff (3 попытки)
- RAW-тикет при полном отказе ИИ — запрос не теряется
- Сохранение упавших запросов в SQLite для ретрая

## Промпты

Все промпты хранятся в папке `prompts/` в формате `.md`:

| Файл | Назначение |
|------|-----------|
| `extraction.md` | Анализ сообщения сейлза, извлечение структурированных данных |
| `dedup.md` | Дедупликация одного запроса против одного тикета |
| `dedup_batch.md` | Батч-дедупликация: один запрос против всего бэклога за раз |
| `scoring.md` | RICE-скоринг и расстановка приоритетов (роль CPO) |

```
│   ├── extraction.md

│   └── scoring.md

```

## Переменные окружения

| Переменная | Обязательная | Описание |
|-----------|:---:|---------|
| `TELEGRAM_TOKEN` | ✅ | Токен Telegram бота (от @BotFather) |
| `LLM_PROVIDER` | ✅ | Основной провайдер: `gemini` / `deepseek` / `openai` / `anthropic` |
| `LLM_API_KEY` | ✅ | API ключ основного провайдера |
| `DEEPSEEK_API_KEY` | — | API ключ DeepSeek (резервный провайдер) |
| `GEMINI_MODEL` | — | Модель Gemini (default: `gemini-2.5-flash`) |
| `DEEPSEEK_MODEL` | — | Модель DeepSeek (default: `deepseek-chat`) |
| `JIRA_URL` | ✅ | URL Jira instance (`https://company.atlassian.net`) |
| `JIRA_EMAIL` | ✅ | Email аккаунта Jira |
| `JIRA_API_TOKEN` | ✅ | API токен Jira |
| `JIRA_PROJECT_KEY` | ✅ | Ключ проекта в Jira (например `TT`) |
| `PRODUCT_STRATEGY` | — | Описание продуктовой стратегии для скоринга |
| `ALLOWED_USERNAMES` | — | Список Telegram username через запятую (пусто = доступ для всех) |
| `CONFIDENCE_THRESHOLD` | — | Порог уверенности для уточнений (default: `0.7`) |
| `DEDUP_DAYS` | — | Окно дедупликации в днях (default: `90`) |
| `MAX_CLARIFICATION_ROUNDS` | — | Максимум раундов уточнений (default: `3`) |
| `JIRA_TIMEOUT` | — | Таймаут запросов к Jira в секундах (default: `10`) |
| `LLM_TIMEOUT` | — | Таймаут запросов к LLM в секундах (default: `30`) |
| `LOG_LEVEL` | — | Уровень логирования (default: `INFO`) |

## RICE-скоринг

| Балл | Приоритет | Действие |
|------|-----------|---------|
| ≥ 250 | 🔴 Highest | Немедленно в спринт |
| 120–249 | 🟠 High | В ближайший квартал |
| 50–119 | 🟡 Medium | В бэклог |
| < 50 | 🟢 Low | Отложить или отклонить |

**Формула:** `Reach × Impact × Confidence × Strategy Fit` (макс. 400)

**Enterprise-множитель:** если клиент Enterprise и `revenue_at_risk ≥ 9` — `reach` принудительно поднимается до 5, `strategy_fit` до 10.
